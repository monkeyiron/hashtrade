# Convex.dev — Hashtrade Usage Guide

This project uses **Convex** as the real-time backend-as-a-service. The frontend is
**Vite + React** (not Next.js). Convex lives at the **monorepo root** in `convex/`.

Use the Exa MCP to fetch `https://docs.convex.dev/quickstart/react` or specific doc pages.

## Project Structure

```
hashtrade/                          ← monorepo root
├── convex/                         ← ALL backend code lives here
│   ├── _generated/                 ← auto-generated by CLI (DO NOT EDIT)
│   ├── schema.ts                   ← database schema (OKLCH-aware, typed)
│   ├── tasks.ts                    ← example query/mutation file
│   └── ...                         ← add more domain files here
├── apps/
│   └── web/
│       ├── src/
│       │   ├── main.tsx            ← ConvexProvider wraps the app here
│       │   └── ...
│       └── .env.local              ← VITE_CONVEX_URL (gitignored)
└── .env.local                      ← CONVEX_DEPLOYMENT (gitignored)
```

> **Rule**: The `convex/` folder MUST be at the **monorepo root** (`c:\ie\hashtrade\convex\`),
> not inside `apps/web/`. The CLI resolves it from where `npx convex dev` is run.

## Frontend Connection Pattern (Vite + React)

The `ConvexProvider` is initialized in `apps/web/src/main.tsx`:

```typescript
import { ConvexProvider, ConvexReactClient } from "convex/react"

const convex = new ConvexReactClient(import.meta.env.VITE_CONVEX_URL as string)

ReactDOM.createRoot(document.getElementById("root")!).render(
  <React.StrictMode>
    <ConvexProvider client={convex}>
      <App />
    </ConvexProvider>
  </React.StrictMode>,
)
```

> **Key**: `VITE_CONVEX_URL` is the env var prefix for Vite (not `NEXT_PUBLIC_CONVEX_URL`).

## Writing Backend Functions

### Schema (`convex/schema.ts`)

Always define a schema for type safety. Use `v` validators from `convex/values`:

```typescript
import { defineSchema, defineTable } from "convex/server"
import { v } from "convex/values"

export default defineSchema({
  // Example table
  listings: defineTable({
    title: v.string(),
    description: v.optional(v.string()),
    price: v.number(),
    currency: v.string(),
    userId: v.string(),            // Clerk userId
    isActive: v.boolean(),
    createdAt: v.number(),         // Unix timestamp
  })
    .index("by_user", ["userId"])
    .index("by_active", ["isActive"]),
})
```

### Query (read data — real-time subscribed)

```typescript
// convex/listings.ts
import { query } from "./_generated/server"
import { v } from "convex/values"

export const getAll = query({
  args: {},
  handler: async (ctx) => {
    return await ctx.db.query("listings")
      .filter((q) => q.eq(q.field("isActive"), true))
      .order("desc")
      .collect()
  },
})

export const getById = query({
  args: { id: v.id("listings") },
  handler: async (ctx, { id }) => {
    return await ctx.db.get(id)
  },
})
```

### Mutation (write data)

```typescript
// convex/listings.ts
import { mutation } from "./_generated/server"
import { v } from "convex/values"

export const create = mutation({
  args: {
    title: v.string(),
    price: v.number(),
    currency: v.string(),
  },
  handler: async (ctx, args) => {
    const identity = await ctx.auth.getUserIdentity()
    if (!identity) throw new Error("Unauthenticated")

    return await ctx.db.insert("listings", {
      ...args,
      userId: identity.subject,
      isActive: true,
      createdAt: Date.now(),
    })
  },
})
```

### Action (external API calls, scheduled tasks)

```typescript
// convex/external.ts
import { action } from "./_generated/server"
import { v } from "convex/values"

export const fetchRates = action({
  args: { currency: v.string() },
  handler: async (_ctx, { currency }) => {
    const res = await fetch(`https://api.exchangerate.host/latest?base=${currency}`)
    return res.json()
  },
})
```

## Using Convex in React Components

```typescript
"use client"  // NOT needed in Vite — this is Next.js only
import { useQuery, useMutation } from "convex/react"
import { api } from "../../convex/_generated/api"  // adjust path from component

// Read (real-time, auto-updates)
function ListingList() {
  const listings = useQuery(api.listings.getAll)  // undefined = loading

  if (listings === undefined) return <div>Loading...</div>

  return (
    <ul>
      {listings.map(({ _id, title, price }) => (
        <li key={_id}>{title} — {price}</li>
      ))}
    </ul>
  )
}

// Write
function CreateListing() {
  const create = useMutation(api.listings.create)

  const handleSubmit = async () => {
    await create({ title: "My Listing", price: 100, currency: "USD" })
  }

  return <button onClick={handleSubmit}>Create</button>
}
```

> **Import path**: Always use `../../convex/_generated/api` (relative from `apps/web/src/`).
> Or configure a path alias `@convex` → `../../convex/_generated` in `vite.config.ts`.

2. mcp_convex-mcp_tables({ deploymentSelector })

## Anti-Patterns — AVOID

| Anti-Pattern | Correct Pattern |
|---|---|
| Putting `convex/` inside `apps/web/` | Keep `convex/` at monorepo root |
| Editing `_generated/` folder manually | Let CLI regenerate it |
| Using `NEXT_PUBLIC_CONVEX_URL` | Use `VITE_CONVEX_URL` (Vite prefix) |
| Running mutations in `useQuery` handlers | Separate queries and mutations |
| Returning sensitive data in public queries | Use `ctx.auth.getUserIdentity()` guard |
| Creating new Convex tables without schema entries | Always define in `schema.ts` first |
| Awaiting `useQuery` result (it's sync) | Check for `undefined` → loading state |

## Checklist Before Implementing Convex Features

- [ ] `convex/schema.ts` has the table defined with validators
- [ ] Query/mutation/action is in the correct `convex/*.ts` file
- [ ] Frontend component checks `=== undefined` for loading state
- [ ] Mutations requiring auth call `ctx.auth.getUserIdentity()` and throw if `null`
- [ ] Indexes are defined for any fields used in `.filter()` or `.order()`
- [ ] `VITE_CONVEX_URL` is set in `apps/web/.env.local`
- [ ] `npx convex dev` is running in a background terminal
- [ ] `_generated/` is NOT committed to git (add to `.gitignore` if needed)
